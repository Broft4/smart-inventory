Этап 1: пользователи, вход, роли, привязка к точке, CRUD пользователей в админке.

Что добавлено:
- вход по логину и паролю
- роли admin / employee
- автосоздание первого администратора:
  логин: admin
  пароль: admin123
- сотрудник привязан к точке и не выбирает ее сам
- админка умеет создавать, редактировать и удалять пользователей
- в админке сохранен блок просмотра ревизий
- в проблемных товарах показывается ФИО сотрудника, который сделал проверку

Как запустить:
1. pip install -r requirements.txt
2. uvicorn app.main:app --reload
3. открыть /login

Важно:
- если у вас осталась старая inventory.db, приложение само пересоздаст несовместимую схему отчетов
- текущая логика ревизии пока работает на мок-данных
- этап выбора категорий сотрудником внутри общей ревизии еще не реализован
